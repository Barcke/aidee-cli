# Lessons

- Public-facing project documentation should avoid repository caveats, internal release readiness notes, and audience filtering language unless the user explicitly wants a maintainer-focused document.
- For bilingual top-level documentation, keep both language files at the repository root when the user asks for equal visibility, and update cross-links in the same edit.
- 面向客户的 README 必须先按源码核对功能边界，不能把内部能力、预留能力或相近模块能力混写成客户可用功能。
- 当用户明确要求中文为主时，根 `README.md` 应优先使用中文文案，而不是默认保留英文主文档。
